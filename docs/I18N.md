# 国际化说明

本文档记录 StockTracker 当前的国际化边界和维护规则。

## 当前支持语言

| Locale | 用途 |
| --- | --- |
| `zh-CN` | 默认 UI 语言 |
| `en-US` | 英文 UI |

语言切换入口位于侧边栏底部，组件为 `components/i18n/LanguageSwitcher.tsx`。用户选择会保存到浏览器 `localStorage`，key 为 `stock-tracker-locale`，并同步更新页面 `<html lang>`。

## 代码入口

| 文件 | 职责 |
| --- | --- |
| `lib/i18n/index.tsx` | `I18nProvider`、`useI18n`、翻译函数、日期和数字 locale、市场标签辅助函数。 |
| `lib/i18n/messages.ts` | Locale 列表、市场/单位/占位符翻译和英文消息表。 |
| `components/i18n/LanguageSwitcher.tsx` | 语言切换 UI。 |
| `app/layout.tsx` | 注入 `I18nProvider`。 |

## Key 策略

当前第一版 i18n 使用“中文源文案作为 key”的过渡策略：

```tsx
const { t } = useI18n()

return <h1>{t('持仓')}</h1>
```

这样做的原因是项目原本 UI 文案以中文为主，首轮改造可以低成本覆盖现有界面，减少大规模命名和迁移风险。

维护时需要注意：

- 用户可见文案必须通过 `t()` 或 i18n helper 输出。
- 新增中文文案时，请在 `lib/i18n/messages.ts` 补充英文翻译。
- 同一句中文如果在不同上下文需要不同英文，应拆成不同文案，或后续迁移为语义 key。
- 当前策略不适合无限扩展语言；如果继续增加语言或翻译团队协作，建议逐步迁移到 `portfolio.todayPnl` 这类稳定语义 key。

## UI 语言与 AI 分析语言

UI 语言和 AI 输出语言是两个独立设置：

- UI 语言：侧边栏底部语言按钮，影响页面文案、日期和数字 locale。
- AI 分析语言：设置页中的“分析语言”，影响 AI 组合分析、标的分析、大盘分析和对话输出。

两者不要强行绑定。用户可能希望英文界面配中文分析，或中文界面配英文分析。

## 新增文案检查

新增或修改用户可见文案时建议检查：

```bash
rg -n "[\\p{Han}]" app components lib --glob '!lib/i18n/messages.ts'
```

该命令会列出仍包含中文的代码位置。并不是所有结果都必须移除，例如：

- 注释。
- AI Prompt。
- 服务端内部日志事件。
- 中文作为翻译 key 的 `t('...')` 调用。

但裸露在 JSX、按钮、placeholder、错误提示里的中文应接入 i18n。
